# MeArm V1.0 工作台 — 首次测试教程

> 更新时间: 2026-06-16

## 0. 准备工作

### 硬件清单
- [ ] Arduino Uno R3（已烧录 `main.ino` / `主程序.txt` 固件）
- [ ] MeArm V1.0 机械臂（4个舵机）
- [ ] USB 数据线（连接电脑和 Uno）
- [ ] 外接电源（推荐 5V 2A 以上，机械臂工作时需要）
- [ ] USB 摄像头（或笔记本自带摄像头，可用 `--cam auto`）

### 接线确认
| 舵机   | Arduino 引脚 |
|--------|-------------|
| 底座   | D11         |
| 左侧   | D10         |
| 右侧   | D9          |
| 夹爪   | D6          |

> ⚠️ **重要**：机械臂舵机必须使用外接电源供电！USB 供电可能导致 Uno 重启或舵机无力。Arduino 和舵机的 GND 要共地。

### 关节限位速查
| 关节 | 范围 | 回零位 |
|------|------|--------|
| base (底座) | 0° – 180° | 90° |
| left (左臂) | 0° – 180° | 90° |
| right (右臂) | 0° – 180° | 90° |
| claw (夹爪) | 0° – 90° | 25° (半开) |

## 1. 连接硬件并确认串口

### 1.1 查找串口号
打开工作台，先不连机械臂，运行以下命令列出可用串口：

```bash
cd .
python workbench_server.py --list-ports
```

或者双击 `run_workbench.bat` 后手动查看。

记住 Arduino 的 COM 端口号（Windows 上一般是 `COM3` 或 `COM4`）。

### 1.2 检查 Arduino 是否正常
用串口工具验证（可选）：
```bash
# 用 Python 快速测试
python -c "
import serial
ser = serial.Serial('COM3', 115200, timeout=2)
import time; time.sleep(2)
ser.reset_input_buffer()
ser.write(b'status\n')
print(ser.readline().decode())
ser.close()
"
```
如果返回类似 `STATUS:base:90,left:90,right:90,claw:25` 说明通信正常。

> ⚠️ 旧固件返回 `STATUS:base:30,left:150,right:150,claw:90` — 说明固件未更新，需要重新烧录。

## 2. 首次启动工作台（分步测试）

### 第一步：仅测试通信（摄像头 + 串口，不动机械臂）

```bash
cd .

# 连接 Arduino，启动工作台
python workbench_server.py --port COM3
```

打开浏览器访问 **`http://localhost:5000`**。

看到工作台页面后，确认以下状态：
- 🔌 **串口** — 应显示绿色圆点（已连接）
- 📷 **摄像头** — 应显示绿色圆点，画面显示在左侧
- 📋 底部日志应看到 `✅ 串口已连接: COM3` 和 `✅ 摄像头已打开`

> 如果笔记本自带摄像头用 `--cam 0` 是黑的，尝试 `--cam auto` 自动搜索。
>
> 如果串口连不上，检查：
> - COM 端口号是否正确
> - Arduino IDE 是否关闭（会占用串口）
> - USB 线是否连接

---

### 第二步：测试单关节（逐个验证）

在工作台的 **"🦾 关节角度"** 面板中，轻轻拖动滑块：

| 关节 | 范围 | 测试方法 | 预期表现 |
|------|------|---------|---------|
| 🔵 底座 | 0-180° | 拖动从 90 到 120 | 机械臂向右旋转 |
| 🟢 左侧 | 0-180° | 拖动从 90 到 70 | 左臂下放 |
| 🟠 右侧 | 0-180° | 拖动从 90 到 70 | 右臂下放 |
| 🟣 夹爪 | 0-90° | 拖动从 90 到 50 | 夹爪半闭合 |

> ⚠️ **注意**：拖动幅度小一些（10-20 度），确认运动方向正确后再加大范围。
>
> ⚠️ **新固件**：底座起始位置在 90°（居中），HOME=90° 为用户视角 0°，±90° 对称。

同时观察 **"🔌 串口监视器"** 面板：
- 蓝色 TX：发出的命令（如 `base:60`）
- 绿色 RX：Arduino 的回复（`ACK:MOVE` → 运动完成后 `ACK:DONE`）

---

### 第三步：测试回零和夹爪

点击控制按钮区域：

1. **🏠 回零位** — 所有关节回到初始位置（base=90, left=90, right=90, claw=25）
2. **✊ 闭合夹爪** — 夹爪闭合到 0 度
3. **🖐 张开夹爪** — 夹爪张开到 90 度

---

### 第四步：测试手势（需要机械臂活动空间足够）

点击手势按钮：

| 手势 | 说明 | 注意事项 |
|------|------|---------|
| 👋 挥手 | base 90↔0 左右摆动 | 确保底座旋转范围内无障碍 |
| 👉 指向 (前) | base居中+手臂下放 | base=90, 爪半合 |
| 👈 指向左 | 指向左边 | base=0, 爪半合 |
| 👉 指向右 | 指向右边 | base=180, 爪半合 |
| 🙂 点头 | 左臂+右臂交替升降 | 幅度适中 |
| 🤗 问候 | 底座+左臂组合 | 最适合展示 |

> 执行手势时顶部状态栏的机械臂指示灯会**黄色闪烁**，手势结束后恢复。

---

### 第五步：测试视觉识别 (HSV 颜色)

1. **🎨 HSV 调节** — 点击此按钮打开弹窗
2. 选择颜色（如 `red`），调整滑块使摄像头画面中**只有你的目标物体是亮的**
3. 点击 **✅ 应用**

观察 **"🎯 检测结果"** 表格：
- 如果识别到物体，会显示颜色、类名、像素坐标、世界坐标(mm)、置信度、来源(hsv/yolo/merged)
- 表格中的 **🎯 抓取** 按钮可触发 pick-and-place 动作

> HSV 调参技巧：
> - H 低/高：控制色相范围（0-180）
> - S 低/高：控制饱和度（颜色纯度），太低=白色也识别，太高=只有纯色
> - V 低/高：控制亮度（环境光线影响）

---

### 第六步：切换摄像头视图

摄像头面板上方有三个标签页：

| 视图 | 用途 |
|------|------|
| **检测标注** | 看检测结果的圆圈+YOLO框+标签 |
| **原始画面** | 看未处理的摄像头画面 |
| **HSV 掩膜** | 看颜色过滤后的二值画面（调试颜色阈值用）|

---

### 第七步：测试 YOLO 物体识别 (有 GPU 时)

1. 确认已安装依赖: `pip install torch ultralytics`
2. 启动工作台，日志应显示 `🔍 YOLO 检测器已就绪 (模型: yolov8s.pt, 设备: cuda:0)`
3. 将日常物品 (杯子/苹果/瓶子/鼠标/键盘/书等) 放在摄像头前
4. 观察画面中物体出现**彩色矩形框** + 中文标签（如 "杯子 0.85"）
5. 语音指令测试: `"抓杯子"` / `"拿苹果"` / `"捡瓶子"` — 机械臂应识别并执行
6. 检测结果表格中 source 列显示 "yolo" 表示来自 YOLO 检测

> 无 GPU 时 YOLO 会回退到 CPU 模式（较慢）或自动跳过，日志会显示原因。

### 第八步：测试 API 视觉回退 (无 GPU 时)

1. 配置 Ollama 视觉模型: `ollama pull llava` (或 `minicpm-v`)
2. 在 `.env` 中设置: `OLLAMA_VISION_MODEL=llava:latest`
3. 启动工作台，日志显示 `☁️ API 视觉检测器已就绪`
4. 将物体放摄像头前，等待约 2-3 秒（API 调用延迟）
5. 检测结果面板应显示 API 返回的物体信息

> 也支持云端视觉 API (Kimi k2.6 等): 设置 `VISION_API_KEY` 环境变量即可。

### 第九步：测试空间记忆

1. 先执行一次抓取操作（语音或按钮）：`"抓红色"`
2. 操作完成后，空间记忆自动记录
3. 语音查询: `"刚才放的是什么"` / `"把刚刚放的东西拿回来"`
4. Web API 查看: 浏览器打开 `http://localhost:5000/api/spatial-memory`
5. 文件查看: `memory/spatial_memory.json`

## 3. 典型问题排查

| 现象 | 可能原因 | 解决方法 |
|------|---------|---------|
| 串口连不上 | IDE 占用 | 关闭 Arduino IDE/串口监视器 |
| 机械臂不动 | 电源不足 | 外接 5V 2A+ 电源 |
| 机械臂不动 (串口通) | 固件未烧录/不匹配 | 重新烧录 `main.ino` |
| YOLO 未加载 | 无 GPU/缺包 | `pip install torch ultralytics` |
| 摄像头画面黑 | 索引错误 | 试试 `--cam auto` 或 `--cam 1` |
| 舵机抖动 | 角度超限 | 控制在 0-180° (claw 0-90°) |
| 识别不到物体 | 光线或颜色 | 用 HSV 调节器重新调 |
| Arduino 返回 `ERR:SYNTAX` | 串口命令格式错 | 重启工作台，检查波特率 |
| Arduino 返回 `ERR:BUF_OVERFLOW` | 发送过快 | 增大 `SAFE_JOINT_DELAY` |
| 运动到一半停止 | 接插件松动 | 检查舵机杜邦线 |
| 手势误触发频繁 | 光照变化 | 调整摄像头角度, 检查 GESTURE_COOLDOWN |
| 语音识别无反应 | 麦克风未接/模型缺失 | 检查麦克风, 确认模型路径 |
| 回零后位置不对 | 固件与配置不一致 | 检查 HOME_ANGLES: base=90, left=90, right=90, claw=25 |
| Kimi 不工作 | API Key/网络 | 检查网络, Kimi 降级到关键词 |
| 空间记忆查询无结果 | 尚未执行过操作 | 先执行一次抓取 |
| pip 安装慢 | 默认 PyPI 源 | `-i https://pypi.tuna.tsinghua.edu.cn/simple` |

## 4. 通信协议速查（Arduino ↔ Python）

```
Python → Arduino:
  base:90,left:70,right:80                 # 移动关节（任意组合）
  home                                     # 回零
  status                                   # 查询状态

Arduino → Python:
  READY                                    # 启动完成
  ACK:MOVE                                 # 收到移动指令
  ACK:DONE                                 # 所有关节到达目标
  ACK:HOME                                 # 回零完成
  STATUS:base:90,left:90,right:90,claw:25   # 当前角度
  ERR:SYNTAX:xxx                           # 命令格式错误
  ERR:UNKNOWN_JOINT:xxx                    # 关节名错误
  ERR:BUF_OVERFLOW                         # 接收缓冲区溢出
```

> ⚠️ STATUS 返回的是新固件的回零位 (base=90, left=90, right=90, claw=25)。如果返回 base=30,left=150,right=150,claw=90 说明 Arduino 运行的是旧固件。

## 5. 常用命令速查

```bash
# 最常用：连接 COM3 端口启动 (默认安全模式)
python workbench_server.py --port COM3

# 自动搜索摄像头
python workbench_server.py --port COM3 --cam auto

# 手机 IP 摄像头
python workbench_server.py --ip-cam http://192.168.x.x:8080/video

# 仅测试视觉（不连机械臂）
python workbench_server.py

# 外接电源全速模式 (关闭安全保护)
python workbench_server.py --port COM3 --unsafe

# 仅关键词模式 (无需 LLM, 断网可用)
python workbench_server.py --port COM3 --no-llm

# 禁用语音识别
python workbench_server.py --port COM3 --no-voice

# 列出可用串口
python workbench_server.py --list-ports

# 启动但不自动打开浏览器
python workbench_server.py --port COM3 --no-browser

# 指定 Web 端口
python workbench_server.py --web-port 8080
```

## 6. 快速自查清单

首次测试时按顺序过一遍：

1. Arduino USB 已连接 ✓
2. 固件已更新 (base=90, left=90, right=90, claw=25) ✓
3. 外接电源已插上 ✓
4. 关闭了 Arduino IDE ✓
5. 知道 COM 端口号（如 COM3）✓
6. 启动命令：`python workbench_server.py --port COM3` ✓
7. 浏览器打开 `http://localhost:5000` ✓
8. 顶部状态灯：串口🟢 摄像头🟢 ✓
9. 拖动底座滑块 (30→80)，机械臂有反应 ✓
10. 摄像头看到画面 (--cam auto 如果黑屏) ✓
11. 放一个红色/绿色物体，检测面板能看到 ✓
12. (可选) YOLO 检测器已就绪，桌面物体有矩形框 ✓
13. (可选) 语音 "你好" → 机械臂回应并做手势 ✓

全部打勾，就可以正常使用了！
