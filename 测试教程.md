# MeArm V1.0 工作台 — 首次测试教程

## 0. 准备工作

### 硬件清单
- [ ] Arduino Uno R3（已烧录 主程序.txt 固件）
- [ ] MeArm V1.0 机械臂（4个舵机）
- [ ] USB 数据线（连接电脑和 Uno）
- [ ] 外接电源（推荐 5V 2A 以上，机械臂工作时需要）
- [ ] USB 摄像头（或笔记本自带摄像头）

### 接线确认
| 舵机   | Arduino 引脚 |
|--------|-------------|
| 底座   | D11         |
| 左侧   | D10         |
| 右侧   | D9          |
| 夹爪   | D6          |

> ⚠️ **重要**：机械臂舵机必须使用外接电源供电！USB 供电可能导致 Uno 重启或舵机无力。Arduino 和舵机的 GND 要共地。

## 1. 连接硬件并确认串口

### 1.1 查找串口号
打开工作台，先不连机械臂，运行以下命令列出可用串口：

```bash
cd .
python workbench_server.py --list-ports
```

或者双击 `run_workbench.bat` 后手动查看。

记住 Arduino 的 COM 端口号（Windows 上一般是 `COM3` 或 `COM4`）。

### 1.2 检查 Arduino 是否正常
用串口工具验证（可选）：
```bash
# 用 Python 快速测试
python -c "
import serial
ser = serial.Serial('COM3', 115200, timeout=2)
import time; time.sleep(2)
ser.reset_input_buffer()
ser.write(b'status\n')
print(ser.readline().decode())
ser.close()
"
```
如果返回类似 `STATUS:base:90,left:90,right:90,claw:73` 说明通信正常。

## 2. 首次启动工作台（分步测试）

### 第一步：仅测试通信（摄像头 + 串口，不动机械臂）

```bash
cd .

# 连接 Arduino，启动工作台
python workbench_server.py --port COM3
```

打开浏览器访问 **`http://localhost:5000`**。

看到工作台页面后，确认以下状态：
- 🔌 **串口** — 应显示绿色圆点（已连接）
- 📷 **摄像头** — 应显示绿色圆点，画面显示在左侧
- 📋 底部日志应看到 `✅ 串口已连接: COM3` 和 `✅ 摄像头已打开`

> 如果串口连不上，检查：
> - COM 端口号是否正确
> - Arduino IDE 是否关闭（会占用串口）
> - USB 线是否连接

---

### 第二步：测试单关节（逐个验证）

在工作台的 **"🦾 关节角度"** 面板中，轻轻拖动滑块：

| 关节 | 测试方法 | 预期表现 |
|------|---------|---------|
| 🔵 底座 | 拖动从 90 到 60 | 机械臂水平旋转 |
| 🟢 左侧 | 拖动从 90 到 120 | 左臂上抬 |
| 🟠 右侧 | 拖动从 90 到 70 | 右臂上抬 |
| 🟣 夹爪 | 拖动从 73 到 50 | 夹爪半闭合 |

> ⚠️ **注意**：拖动幅度小一些（10-20 度），确认运动方向正确后再加大范围。

同时观察 **"🔌 串口监视器"** 面板：
- 蓝色 TX：发出的命令（如 `base:60`）
- 绿色 RX：Arduino 的回复（`ACK:MOVE` → 运动完成后 `ACK:DONE`）

---

### 第三步：测试回零和夹爪

点击控制按钮区域：

1. **🏠 回零位** — 所有关节回到初始位置（90, 90, 90, 73）
2. **✊ 闭合夹爪** — 夹爪闭合到 10 度
3. **🖐 张开夹爪** — 夹爪张开到 73 度

---

### 第四步：测试手势（需要机械臂活动空间足够）

点击手势按钮：

| 手势 | 说明 | 注意事项 |
|------|------|---------|
| 👋 挥手 | base 左右摆动 | 确保底座旋转范围内无障碍 |
| 👉 指向 | 左臂抬起+右臂收起 | 幅度较大 |
| 🙂 点头 | 左臂+右臂交替 | 幅度适中 |
| 🤗 问候 | 底座+左臂组合 | 最适合展示 |

> 执行手势时顶部状态栏的机械臂指示灯会**黄色闪烁**，手势结束后恢复。

---

### 第五步：测试视觉识别

1. **🎨 HSV 调节** — 点击此按钮打开弹窗
2. 选择颜色（如 `red`），调整滑块使摄像头画面中**只有你的目标物体是亮的**
3. 点击 **✅ 应用**

观察 **"🎯 检测结果"** 表格：
- 如果识别到物体，会显示颜色、像素坐标、世界坐标(mm)
- 表格中的 **🎯 抓取** 按钮可触发 pick-and-place 动作

> HSV 调参技巧：
> - H 低/高：控制色相范围（0-180）
> - S 低/高：控制饱和度（颜色纯度），太低=白色也识别，太高=只有纯色
> - V 低/高：控制亮度（环境光线影响）

---

### 第六步：切换摄像头视图

摄像头面板上方有三个标签页：

| 视图 | 用途 |
|------|------|
| **检测标注** | 看检测结果的圆圈+标签 |
| **原始画面** | 看未处理的摄像头画面 |
| **HSV 掩膜** | 看颜色过滤后的二值画面（调试颜色阈值用）|

## 3. 典型问题排查

| 现象 | 可能原因 | 解决方法 |
|------|---------|---------|
| 串口连不上 | IDE 占用 | 关闭 Arduino IDE/串口监视器 |
| 机械臂不动 | 电源不足 | 外接 5V 2A+ 电源 |
| 舵机抖动 | 角度超限 | 滑块控制在 30-150 范围内 |
| 识别不到物体 | 光线或颜色 | 用 HSV 调节器重新调 |
| Arduino 返回 `ERR:SYNTAX` | 串口命令格式错 | 重启工作台，检查波特率 |
| 摄像头画面黑/无 | 索引错误 | 试试 `--cam 1` |
| 运动到一半停止 | 接插件松动 | 检查舵机杜邦线 |

## 4. 通信协议速查（Arduino ↔ Python）

```
Python → Arduino:
  base:90,left:120,right:80             # 移动关节（任意组合）
  home                                   # 回零
  status                                 # 查询状态

Arduino → Python:
  READY                                  # 启动完成
  ACK:MOVE                               # 收到移动指令
  ACK:DONE                               # 所有关节到达目标
  ACK:HOME                               # 回零完成
  STATUS:base:85,left:90,right:92,claw:50   # 当前角度
  ERR:SYNTAX:xxx                         # 命令格式错误
  ERR:UNKNOWN_JOINT:xxx                  # 关节名错误
  ERR:BUF_OVERFLOW                       # 接收缓冲区溢出
```

## 5. 常用命令速查

```bash
# 最常用：连接 COM3 端口启动
python workbench_server.py --port COM3

# 串口 + 指定摄像头索引
python workbench_server.py --port COM3 --cam 1

# 仅测试视觉（不连机械臂）
python workbench_server.py

# 列出可用串口
python workbench_server.py --list-ports

# 仅运行 HSV 标定（不启动工作台）
python -m workbench_server --calibrate

# 启动但不自动打开浏览器
python workbench_server.py --port COM3 --no-browser
```

## 6. 快速自查清单

首次测试时按顺序过一遍：

1. Arduino USB 已连接 ✓
2. 外接电源已插上 ✓
3. 关闭了 Arduino IDE ✓
4. 知道 COM 端口号（如 COM3）✓
5. 启动命令：`workbench_server.py --port COM3` ✓
6. 浏览器打开 `http://localhost:5000` ✓
7. 顶部状态灯：串口🟢 摄像头🟢 ✓
8. 拖动底座滑块，机械臂有反应 ✓
9. 摄像头看到画面 ✓
10. 放一个红色/绿色物体，检测面板能看到 ✓

全部打勾，就可以正常使用了！
