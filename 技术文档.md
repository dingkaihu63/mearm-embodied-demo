## 1. 项目概述

MeArm 工作台是一个桌面机械臂的全功能**具身智能**交互系统。通过 **摄像头 + 麦克风 + 大语言模型 + 语音合成**，让用户可以用自然语言、语音、手势三种方式控制机械臂，并获得自然的语音反馈。

**硬件**: Arduino Uno R3 + MeArm V1.0 机械臂套件 (4 舵机) + USB 摄像头 + 麦克风

**软件**: Python 3.10 上位机 + Arduino C++ 固件 + Web 前端

**定位**: 低成本 (~200 元) 的具身智能实验平台 — 「感知 → 推理 → 行动」全栈闭环。默认使用本地 Ollama，可配置云端 API。

## 2. 技术路线

```
用户输入 (文本/语音/手势)
    │
    ▼
感知层: Vosk双语语音 / MediaPipe手势 / YOLOv8s+HSV+API三层视觉
    │
    ▼
决策层: 关键词30条规则 → Ollama/云端LLM (可选视觉多模态) — 两层回退
    │  └── 空间记忆上下文注入
    ▼
规划层: 逆运动学求解 + LLM增强策略规划
    │
    ▼
执行层: Arduino串口通信 → 舵机运动
    │
    ▼
反馈层: Edge-TTS语音播报 + Web实时状态推送
    │
    ▼
学习层: 交互记忆 + 空间记忆 → 用户画像 → LLM提示词增强 (闭环)
```

### 2.1 技术栈选型理由

| 环节 | 方案 | 选择理由 |
|------|------|---------|
| 语音识别 | Vosk 离线 | 无需联网，中英文双模型并行评分 |
| 手势识别 | MediaPipe Hands | Google 官方方案，21点关键点，多帧确认+迟滞 |
| 物体检测 | YOLOv8s + HSV + API 三层 | GPU加速80类检测 / 颜色始终运行 / 无GPU时Ollama视觉回退 |
| 多模态理解 | Ollama视觉 / 云端视觉API | 图片+文本联合推理，纠错 Vosk 短词误识别 |
| 语音合成 | Edge-TTS | 微软免费神经网络语音，自然度高 |
| LLM 文本 | Ollama (qwen2.5:7b) 默认 | 本地运行，完全离线，中文友好 |
| LLM 云端 | 云端 API 自适应 (DeepSeek/Kimi等) | 设置环境变量自动切换，OpenAI 兼容 |
| Web | Flask + SocketIO | 轻量，实时双向通信 |
| 串口 | PySerial | Python 标准串口库 |
| 自学习 | 本地 TF-IDF + JSON | 零外部依赖，轻量离线 |
| 空间记忆 | 本地 JSON 持久化 | 100条记录，自动注入 LLM 上下文 |

## 4. 各子系统实现

### 4.1 视觉管线 (`mearm_controller/vision.py` + `vision_yolo.py` + `vision_api.py`)

**三层检测架构** (从快到慢、从本地到远端):

| 层级 | 方法 | 说明 | 帧率 | GPU需求 |
|------|------|------|------|---------|
| 1️⃣ YOLOv8s | 深度学习 80 类 | CUDA 推理 ~5ms, 每 3 帧一次 | ~10 FPS | ✅ |
| 2️⃣ HSV 颜色 | OpenCV 阈值分割 | 红/绿/蓝/黄四色, 每帧运行 | ~30 FPS | ❌ |
| 3️⃣ API 回退 | Ollama/云端视觉 | 无GPU时每 15 帧调用一次 | ~0.5 FPS | ❌ |

**YOLOv8s**: 首次运行自动下载 (~22MB)。40 个中文类名映射覆盖常见桌面物体。

**API 回退**: YOLO 不可用时，通过 `LLMIntentParser.parse_with_image()` 调用视觉模型。支持 Ollama 视觉 (如 llava) 和云端视觉 API (需设置 `VISION_API_KEY`)。

**HSV 流程** (始终运行): BGR→HSV→阈值分割→形态学去噪→轮廓检测→单应性变换→世界坐标。

### 4.2 语音识别 (`mearm_controller/voice_listener.py`)

**方法**: Vosk 离线双模型并行，综合评分取最优。

**稳定性**:
- 语言分阈值: 中文 ≥0.35 / 英文 ≥0.45
- 综合评分: 置信度 + 文本长度加成 + 中文偏好 (+0.10)
- 去重窗口 2s
- 模型路径通过环境变量配置: `VOSK_MODEL_CN` / `VOSK_MODEL_EN`

**默认模型**: `models/vosk-model-small-cn/` (~44MB) + `models/vosk-model-small-en-us-0.15/` (~41MB)

### 4.3 手势识别 (`mearm_controller/gesture_recognizer.py`)

**方法**: MediaPipe HandLandmarker，21 关键点。

**稳定性机制**:
- 多帧确认: 同一手势需连续 5 帧
- 退出迟滞: 连续 8 帧 "none" 才重置
- 挥手: 20 帧历史 / ≥5 次穿越
- 冷却 3s
- MediaPipe 检测阈值 0.8, 跟踪 0.7

**手势分类**: open_palm / fist / pointing / thumbs_up / peace / wave → LLM 决策动作

### 4.4 LLM 意图解析 (`mearm_controller/llm_parser.py`)

**提供者自适应**: 无需修改代码，通过环境变量切换。

```
┌─ LLM 文本提供者 ─────────────────────────────┐
│ LLM_API_KEY 已设置 → 云端 API (DeepSeek等)    │
│ LLM_API_KEY 未设置 → Ollama (qwen2.5:7b)     │
└──────────────────────────────────────────────┘

┌─ 视觉多模态提供者 (可选) ────────────────────┐
│ VISION_API_KEY 已设置 → 云端视觉 (Kimi等)     │
│ OLLAMA_VISION_MODEL 已设置 → Ollama 视觉      │
│ 均未设置 → 纯文本模式                        │
└──────────────────────────────────────────────┘
```

**两层意图解析**:

**语音/手势**: 关键词匹配 (30条) → LLM (带视觉多模态) → LLM (纯文本)

**文本**: 关键词匹配 (30条) → LLM (纯文本)

**关键词规则库** (30 条, 13 大类):

| # | 类别 | 示例 | action |
|---|------|------|--------|
| 1 | 停止/暂停 | 停止、停、stop | say |
| 2 | 回零/复位 | 回零、home、reset | home |
| 3 | 夹爪控制 | 张开/闭合、open/close | claw_open/close |
| 4 | 问候/社交 | 你好、谢谢、再见 | gesture/say |
| 5 | 手势触发 | 挥手/点头/指向/问候 | gesture |
| 6 | 底座旋转 | 左转/右转 | move_joint(base) |
| 7 | 手臂升降 | 抬臂/举高/放低 | move_joint(lift) |
| 8 | 肘部伸缩 | 前伸/后缩 | move_joint(elbow) |
| 9 | 速度控制 | 快点/慢点 | say |
| 10 | 状态查询 | 状态/看到什么 | say |
| 11 | 确认/否定 | 好的/不要/取消 | say |
| 12 | 空间指示 | 刚刚放的/拿回来 | pick_and_place |
| 13 | 帮助 | 帮助/怎么用 | say |

**颜色映射**: 28 个中英文双向。**抓取动词**: 13 个。

**JSON 输出**: `{action, color, class_name, gesture, joint, direction, message, confidence}`

7 种动作类型: pick_and_place / gesture / home / say / claw_open / claw_close / move_joint

**API 调用参数**:
- max_tokens: 512 (意图解析) / 60 (语音回复)
- timeout: 15s (Ollama 本地可能较慢)
- temperature: 0.1 (意图) / 0.8 (回复)
- 视觉模型 temperature: `VISION_API_TEMPERATURE` (默认 1.0, 兼容 Kimi k2.6)

### 4.5 空间记忆 (`mearm_controller/spatial_memory.py`)

每次 pick-and-place 操作自动记录: 物体描述、颜色、类名、抓取坐标、放置坐标、时间戳。

**能力**:
- `get_context_text()` → 自动注入 LLM prompt
- 上下文查询: last_placed() / find_by_color() / find_by_desc()
- 空间引用关键词: "刚刚放的" / "拿回来" / "XX在哪"
- 存储: `memory/spatial_memory.json` (最多 100 条)

### 4.6 逆运动学 (`mearm_controller/arm_ik.py` + `ik_llm.py`)

**解析解 + LLM增强三层降级**: 解析解 → LLM推理替代坐标 → 15%缩回。

**起始位置** (2026-06-12): base=30° / left=150° / right=150° / claw=90°

**关节限位**: base/left/right 30°–150°, claw 0°–90°

### 4.7 串口通信 (`mearm_controller/arm_serial.py`)

文本行协议, 115200bps。ACK:MOVE → ACK:DONE 两阶段确认。默认安全模式逐关节发送。

### 4.8 语音合成 (`mearm_controller/speaker.py`)

Edge-TTS (zh-CN-XiaoxiaoNeural) → pygame 播放。回退 pyttsx3 (离线)。

### 4.9 自学习库 (`mearm_learner/`)

TF-IDF + JSON: 交互记忆 → 用户画像 → Prompt 增强。10 条即可生效。

### 4.10 Web 前端 (`templates/workbench.html`)

原生 HTML/CSS/JS + SocketIO。三路 MJPEG、关节滑块、YOLO+HSV检测面板、串口监视、HSV在线调节、手势开关。

## 6. 制作思路与设计决策

### 6.1 为什么默认本地 LLM

开源项目应该降低使用门槛。Ollama + qwen2.5:7b 完全离线、免费、中文友好。同时通过 `.env` 保留云端 API 选项，满足不同用户的需求。

### 6.2 为什么 LLM 提供者自适应

避免维护多套代码路径。`LLMIntentParser` 在初始化时检测环境变量，自动选择 Ollama 或云端 API，上层调用完全统一。

### 6.3 为什么安全模式是默认的

USB 供电 ~500mA, 四舵机同时启动 > 1A → Uno 重启。安全模式逐关节发送。外接电源后可 `--unsafe` 关闭。

### 6.4 模块化架构

从单文件重构为 16 模块的 `mearm_controller/` 包。模块间通过 `shared_state` 通信。

### 6.5 自学习用本地方案

零依赖 TF-IDF + JSON，10 条交互即可产出画像，人类可读。

### 6.6 所有配置通过环境变量

无硬编码 API Key。`.env` 文件 + CLI 参数覆盖。详见 `.env.example`。

## 7. 实际效果与已知局限

### 7.1 实测表现

| 功能 | 实测效果 | 备注 |
|------|---------|------|
| 文本指令 | 关键词 <1ms / Ollama 1-3s / 云端 1-3s | 准确率 >90% |
| 语音识别 | 安静环境约 70-80% | Vosk 小模型, 视觉纠错提升 |
| 手势识别 | 室内正常光线约 85-90% | 5帧确认减少误触发 |
| YOLO 检测 | 80 类, CUDA ~5ms | 每 3 帧一次 |
| API 视觉回退 | Ollama视觉 2-5s / 云端 1-3s | 无 GPU 时 |
| 机械臂定位 | ±5mm | 无编码器 |
| 语音播报 | Edge-TTS 自然, 0.5-1s | 需联网 |
| 空间记忆 | <1ms 查询 | 最多 100 条 |

### 7.2 已知局限

无位置反馈、单目深度不精确、语音受噪音影响、Ollama 本地对硬件有要求 (建议 16GB+ RAM)、无抓取力反馈、无碰撞检测。
