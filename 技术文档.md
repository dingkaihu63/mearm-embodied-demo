## 1. 项目概述

MeArm 工作台是一个桌面机械臂的全功能交互系统。通过 **摄像头 + 麦克风 + 大语言模型 + 语音合成**，让用户可以用自然语言、语音、手势三种方式控制机械臂，并获得自然的语音反馈。

**硬件**: Arduino Uno R3 + MeArm V1.0 机械臂套件 (4 舵机) + USB 摄像头 + 麦克风

**软件**: Python 3.10 上位机 + Arduino C++ 固件 + Web 前端

```
用户输入 (文本/语音/手势)
    │
    ▼

    │
    ▼
决策层: DeepSeek LLM 统一意图解析 (含关键词规则回退)
    │
    ▼
规划层: 逆运动学求解 + LLM增强策略规划
    │
    ▼
执行层: Arduino串口通信 → 舵机运动
    │
    ▼
反馈层: Edge-TTS语音播报 + Web实时状态推送
    │
    ▼
学习层: 交互记忆 → 用户画像 → LLM提示词增强 (闭环)
```

| 环节 | 方案 | 选择理由 |

## 4. 各子系统实现

### 4.1 视觉管线 (`mearm_controller/vision.py` + `vision_yolo.py` + `vision_api.py`)

**三层检测架构**:

| 层级 | 方法 | 说明 | 速度 |
|------|------|------|------|
| 1️⃣ YOLO | YOLOv8s 深度学习 | 80 类通用物体, 本地 CUDA 推理 | 每 3 帧 |
| 2️⃣ HSV | OpenCV 颜色检测 | 红/绿/蓝/黄四色, 始终运行 | 每帧 |
| 3️⃣ API | Ollama 视觉模型 | 云端回退 (llava/minicpm-v), 无 GPU 时 | 每 15 帧 |

**YOLOv8s**: ultralytics 提供的预训练模型 (22MB), 可识别杯子/瓶子/苹果/书/鼠标等 80 类常见物体。推理在 RTX 5060 上约 5ms/帧。

**API 回退**: 当 YOLO 不可用时, 将摄像头画面编码为 JPEG base64, 调用 Ollama 视觉模型做物体检测, 返回结构化的物体列表和位置描述。

**HSV 流程** (保留):
1. BGR → HSV 转换 → `cv2.inRange` 阈值分割
2. 形态学开闭运算去噪 → 轮廓检测 (面积 ≥800px²)
3. 重心计算 + 单应性变换 → 世界坐标 (mm)

### 4.2 语音识别 (`mearm_controller/voice_listener.py`)

**方法**: Vosk 离线语音识别, 中英文双模型并行

**流程**:
1. PyAudio 16kHz 单声道采集
2. 同时送入两个 KaldiRecognizer (中文 vosk-model-small-cn, 英文 vosk-model-small-en-us-0.15)
3. 取置信度较高的识别结果
4. 结果通过 `queue.Queue` 送入指令处理管道

**局限性**: Vosk 小模型准确率有限, 尤其在噪音环境下。中文模型约 66MB, 英文约 68MB。

### 4.3 手势识别 (`mearm_controller/gesture_recognizer.py`)

**方法**: MediaPipe HandLandmarker (Tasks API)

**流程**:
1. 从摄像头画面提取 RGB 帧
2. MediaPipe 检测 21 个手部关键点 (x, y, z)
3. 基于手指关节几何关系判断伸展/弯曲:
   - 指尖到 MCP 距离 > PIP 到 MCP 距离 × 1.2 → 伸展
   - 拇指特殊处理: 横向距离与手腕比较
4. 手指伸展数 → 手势分类:

| 伸展数 | 手指组合 | 手势 |
|--------|---------|------|
| 5 | 全部 | open_palm |
| 0 | 无 | fist |
| 1 | 食指 | pointing |
| 1 | 拇指 | thumbs_up |
| 2 | 食指+中指 | peace |

5. 挥手检测: 跟踪手腕 x 坐标在 15 帧内的方向变化, ≥3 次穿越判定为 wave
6. 冷却机制: 同手势 2 秒内不重复触发

**局限性**: 依赖光照条件, 手势识别准确率约 85-90% (室内正常光线)。强背光或手部遮挡会明显下降。

### 4.4 LLM 意图解析 (`mearm_controller/llm_parser.py`)

**方法**: DeepSeek API (deepseek-v4-flash 模型)

**流程**:
1. 构造 system prompt, 定义机械臂能力、约束、输出格式
2. 将用户输入 + 当前可见物体列表发送给 DeepSeek
3. 解析返回的 JSON: `{action, color, gesture, message, confidence}`
4. 四种动作类型:
   - `pick_and_place`: 抓取指定颜色物体
   - `gesture`: 执行预设手势序列
   - `home`: 回零
   - `say`: 仅语音回复

**回退策略**: LLM 不可用 (网络/API问题) 时, 使用关键词匹配规则:
- "抓/拿/捡 + 颜色词" → pick_and_place
- "挥手/点头/问候" → gesture
- "回零/home" → home

**API 调用参数**:
- model: deepseek-v4-flash
- max_tokens: 256 (意图解析) / 60 (语音回复生成)
- temperature: 0.1 (意图解析, 需确定性) / 0.8 (语音回复, 需多样性)
- timeout: 10s / 5s

### 4.5 逆运动学 (`mearm_controller/arm_ik.py` + `ik_llm.py`)

**解析解 (ArmIK)**:
- 将 MeArm 平行四边形机构等效为单连杆 (L1+L2=150mm)
- 底座角度: `arctan2(x, y)`
- 肩部角度: `arctan2(dz, reach_2d)` (等效连杆与水平面夹角)
- 肘部: 通过 `ELBOW_SHOULDER_RATIO` 联动 (默认 1.0)

**LLM 增强 (IKLLMEnhancer)**:

1. **可达性降级**: 目标超出范围时:
   - 第1层: 解析解 → 成功返回
   - 第2层: LLM 推理最近可达替代坐标 → 重试
   - 第3层: 沿方向缩回 15% → 最终尝试

2. **策略规划**: LLM 根据物体位置决定:
   - 接近高度、抓取高度、搬运高度
   - 是否先推近 (远处物体)
   - 放置位置坐标

3. **自标定**: 分析历史执行偏差, LLM 建议 `ELBOW_OFFSET` 修正值

**参数**:
- L1 = L2 = 75mm
- H = 55mm (肩关节高度)
- 摄像头: 底座上方 240mm, 水平朝前

### 4.6 串口通信 (`mearm_controller/arm_serial.py`)

**协议**: 文本行协议, 换行符分隔, 115200 bps

**两阶段确认**:
- `ACK:MOVE`: Arduino 收到指令后立即回复 (命令已接收)
- `ACK:DONE`: 所有舵机到达目标位置后回复 (运动已完成)

**安全模式**: USB 供电时逐关节发送, 间隔 80ms, 避免多舵机同时启动导致的瞬时电流过大。

### 4.7 语音合成 (`mearm_controller/speaker.py`)

**主方案**: Edge-TTS (微软免费神经网络 TTS)
- 语音: zh-CN-XiaoxiaoNeural (女声)
- 输出: MP3 字节流
- 播放: pygame.mixer 播放临时 MP3 文件

**回退方案**: pyttsx3 (Windows SAPI5 本地 TTS)
- 自动选择中文语音引擎
- 音质较差但完全离线

### 4.8 自学习库 (`mearm_learner/`)

**交互记忆 (`memory.py`)**:
- 数据结构: `Interaction` dataclass (时间戳、输入类型、原始文本、LLM意图、执行动作、成功标志)
- 存储: JSON 文件 (`memory/interactions.json`)
- 检索: 基于 TF-IDF 的余弦相似度, 支持中英文混合分词
- 中文分词: 2-gram + 单字过滤停用词 (不依赖 jieba)

**技能学习 (`learner.py`)**:
- 最小数据量: 10 条交互
- 分析维度: 偏好颜色、偏好手势、常用命令、成功率、主要输入方式
- 输出: 用户画像摘要 + LLM prompt 增强文本

**提示词适配 (`adapter.py`)**:
- 将用户画像注入 LLM system prompt
- 带缓存: 画像不变时不重复生成
- 少样本示例: 选取最近成功案例作为 few-shot

### 4.9 Web 前端 (`templates/workbench.html`)

**技术**: 原生 HTML/CSS/JS + SocketIO 客户端

**功能模块**:
- 三路 MJPEG 摄像头画面 (检测标注/原始/HSV掩膜)
- 四关节滑块 (base/left/right/claw) + 实时角度仪表条
- 物体检测结果表格 (颜色/坐标/操作按钮)
- 快捷控制按钮 (回零/夹爪/手势/颜色抓取/HSV调节/急停)
- 串口监视器 (TX蓝色/RX绿色)
- 可展开事件日志 (最近 200 条)
- 语音指令输入框 + 语音开关
- 语音回复 Toast 弹窗
- HSV 颜色范围在线调节弹窗
- 手势识别开关

## 6. 制作思路与设计决策

### 6.1 为什么用 LLM 统一决策

机械臂的传统控制方式是规则匹配 (if-else)。但自然语言命令变化无穷 ("把那个红的拿过来"/"grab the red one"/"红色的那个")，规则覆盖不全。

采用 LLM 统一决策的好处:
- 同一套 prompt 处理文本、语音、手势三种输入
- 自动理解同义表达，不需要穷举规则
- 可以生成有变化的自然语音回复 (而非固定话术)

代价:
- 依赖网络 (DeepSeek API)
- 延迟约 500ms-2s
- 有 API 费用

为此设计了关键词规则回退，确保断网时基本功能可用。

### 6.2 为什么安全模式是默认的

Arduino Uno 的 5V 引脚通过 USB 供电时只能提供约 500mA。四个 SG90 舵机同时运动会超过这个限制，导致 Uno 重启或舵机无力。

安全模式逐关节发送指令 (间隔 80ms)，将瞬时电流控制在安全范围内。外接 5V 2A+ 电源后可通过 `--unsafe` 关闭。

### 6.3 为什么用模块化架构

最初所有代码在单个 2000 行的 `workbench_server.py` 中。随着功能增加 (手势识别、自学习、LLM增强IK)，维护变得困难。

重构为 12 个模块的 `mearm_controller/` 包:
- 每个模块职责单一 (视觉/语音/手势/LLM/串口/IK 各自独立)
- 模块间通过 `shared_state` 全局实例通信
- 入口脚本 `workbench_server.py` 保持兼容，`run_workbench.bat` 无需修改

### 6.4 为什么自学习用本地方案

云端方案 (如向量数据库) 需要额外服务和费用。本地 TF-IDF + JSON 方案:
- 零依赖，不增加部署复杂度
- 中文 2-gram 分词在短指令场景下够用
- 10 条交互即可产出有意义的用户画像
- JSON 文件人类可读，方便调试

局限性: 无法做真正的语义理解 (如 "拿那个圆的" 和 "抓取球形物体" 不会被识别为相似)。

### 6.5 关节命名

Arduino 固件使用 `base/left/right/claw` (反映物理舵机位置)，早期 Python 代码使用 `base/shoulder/elbow/claw` (反映运动学模型)。现已统一为 Arduino 的命名，避免通信时的映射错误。

## 7. 实际效果与已知局限

### 7.1 实测表现

| 功能 | 实测效果 | 备注 |
|------|---------|------|
| 文本指令 | 响应时间 1-3s，准确率 >90% | 取决于 DeepSeek API 延迟 |
| 语音识别 | 安静环境准确率约 70-80% | Vosk 小模型，噪音下下降明显 |
| 手势识别 | 室内正常光线准确率约 85% | 背光/遮挡会下降 |
| 颜色检测 | 红/绿色检测稳定，蓝/黄色受光照影响大 | HSV 在线可调 |
| 机械臂定位 | 解析 IK 精度约 ±5mm | 无闭环反馈 (无编码器) |
| LLM IK降级 | 超出范围时可找到替代方案 | 依赖 API，离线不可用 |
| 语音播报 | Edge-TTS 自然度高，延迟 0.5-1s | 需联网 |
| 自学习 | 10条后产出画像，50条后较准确 | 短指令场景够用 |

### 7.2 已知局限

1. **无位置反馈**: SG90 舵机无编码器，无法确认实际到达位置。通信协议依赖 Arduino 的 ACK:DONE 但这是基于步进计数而非传感器。

2. **视觉深度估算不精确**: 单目摄像头水平拍摄，深度估计依赖单应性变换和像素位置启发式，无法做到毫米级精度。

3. **语音识别受噪音影响**: Vosk 小模型在风扇/空调噪音下准确率显著下降。可考虑升级为大模型 (vosk-model-cn-0.22, ~1.4GB)。

4. **LLM 延迟不可控**: DeepSeek API 偶有排队，最长延迟可达 3-5 秒。

5. **抓取成功率**: 无夹爪力反馈，小/薄/光滑物体可能抓取失败。

6. **自学习冷启动**: 需要 ≥10 条交互才开始生效，初期无个性化能力。

7. **无碰撞检测**: 运动学模型不考虑自碰撞和工作空间障碍物。

### 7.3 与商业产品对比

| 维度 | 本项目 | 商业产品 (如 Dobot) |
|------|--------|---------------------|
| 成本 | ~200元 (套件+Arduino) | 3000-20000元 |
| 精度 | ±5mm (无编码器) | ±0.2mm (步进电机+编码器) |
| 交互方式 | 自然语言/语音/手势 | 图形界面/API |
| 智能程度 | LLM 驱动, 可自主学习 | 规则/脚本驱动 |
| 可靠性 | 实验级别 | 工业/教育级别 |

本项目定位是**低成本实验平台**，在交互智能性上有创新，但在机械精度和可靠性上与商业产品有本质差距。
